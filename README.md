# Technology Stack
## Frontend
React.js ทำหน้าที่เป็น Frontend ของระบบ ใช้สร้างหน้าเว็บไซต์ รับข้อมูลจากผู้ใช้งาน และแสดงผลข้อมูลการวิเคราะห์ความเสี่ยง เช่น แผนที่และกราฟ 

ทำอะไรได้บ้าง
* สร้างหน้าเว็บแบบ Interactive
จัดการข้อมูลที่เปลี่ยนแปลงแบบ Real-time
สร้าง Component ที่นำกลับมาใช้ซ้ำได้
เชื่อมต่อ API เพื่อดึงข้อมูลจาก Backend
แสดงข้อมูลแผนที่ ตาราง กราฟ และฟอร์ม

ข้อดีของ React

* สามารถสร้าง Component และนำกลับมาใช้ซ้ำได้ ช่วยลดการเขียนโค้ดซ้ำ ทำให้โค้ดเป็นระเบียบและแก้ไขได้ง่าย
* มี Virtual DOM ช่วยเพิ่มประสิทธิภาพในการอัปเดตและแสดงผลหน้าเว็บ
* สนับสนุนการเขียนโค้ดแบบ Declarative ทำให้การพัฒนา User Interface ง่ายและรวดเร็วขึ้น

ข้อเสียของ React

* มีการ อัปเดตไลบรารีและวิธีการใช้งานอยู่บ่อยครั้ง ทำให้นักพัฒนาต้องเรียนรู้และปรับตัวตามการเปลี่ยนแปลงอยู่เสมอ
* หากไม่ติดตามการอัปเดต อาจทำให้เกิด ปัญหาในการพัฒนาและดูแลระบบ ได้
## Map Library
Leaflet.js ใช้สร้างแผนที่ Interactive และแสดงข้อมูล เช่น Marker และ Heatmap 
Leaflet ได้รับการออกแบบโดยคำนึงถึงความเรียบง่ายประสิทธิภาพและการใช้งานมันทำงานได้อย่างมีประสิทธิภาพบนแพลตฟอร์มเดสก์ท็อปและมือถือหลักๆ ทั้งหมดสามารถขยายขีดความสามารถด้วยปลั๊กอินมากมายมีAPI ที่สวยงามใช้งานง่ายและมีเอกสารประกอบอย่างดี และซอร์สโค้ดที่เรียบง่ายอ่านง่าย ทำให้การมีส่วนร่วมในการพัฒนาเป็นเรื่องสนุก
- ทำอะไรได้บ้าง
แสดงแผนที่บนเว็บไซต์
ปักหมุด (Marker)
วาดเส้น (Polyline)
วาดพื้นที่ (Polygon)
แสดง Popup
คำนวณตำแหน่งพิกัด
รองรับ GeoJSON
## Map Data 
OpenStreetMap เป็นแหล่งข้อมูลแผนที่ ส่วน Leaflet.js เป็น Library ที่นำข้อมูลแผนที่มาแสดงผลบนเว็บไซต์
- ทำอะไรได้บ้าง
ให้ข้อมูลถนน
ข้อมูลแม่น้ำ
ข้อมูลอาคาร
ข้อมูลหมู่บ้าน
ข้อมูลสถานที่สำคัญ
## FastAPI
 FastAPI ทำหน้าที่เป็น Backend ที่รับ Request จาก React.js ประมวลผลข้อมูล เชื่อมต่อ Database และ AI แล้วส่งผลลัพธ์กลับไปยัง Frontend
- ทำอะไรได้บ้าง
รับข้อมูลจาก Frontend
ประมวลผลข้อมูล
ติดต่อฐานข้อมูล
เรียกใช้งาน AI Model
ส่งข้อมูลกลับไปยัง Frontend
สร้าง REST API
